文字：
長度設定後,若中文字則需要除以3
CHAR(固定長度)假設設定20,輸入5個字,撈取出來是20個字其餘補空白(用來設定固定號碼的.如電話號碼、身分證字號)

VACHAR(可變動長度)設定20,輸入5個字,撈取出來也是5個字(用來設定email、地址等)
cname中文姓名長度需輸入135(台灣更姓名最高45字需x3)
adderss地址輸入300(共100字中文)
tel電話輸入

INT數值(資料庫只接受數字)
BIGINT(可存放較多數字值)
若要建立tel手機號碼時:
優點：1、減少資料存入的錯誤 
2、減少占用硬碟的空間
缺點:1、分機號碼無法輸入,區域號碼也補不進去
2、第一個字輸入0,資料型態會吃掉0

若不清楚資料型態則使用VACHAR，但CHAR資料型態在撈出時速度稍微快點，VACHAR在輸出時將後面多於空白刪除時完整資料,

索引設定：

CREATE TABLE `mydb`.`userinfo` ( `cname` VARCHAR(135) NOT NULL , `address` VARCHAR(300) NOT NULL , `tel` VARCHAR(20) NOT NULL , PRIMARY KEY (`cname`)) ENGINE = InnoDB;(ENGINE = InnoDB;只有SQL獨有)

主索引、主鍵、Primary Key、PK

若沒有設定主索引時,當兩筆資料型態完全一樣,資料庫再進行刪除或更改就鎖定指定值,就無法刪除更改,可以手動建立一個流水號

若拆成兩個或更多資料表時,其中關連線、查詢軌跡須了解
查詢軌跡 = 1、查詢目標條件 2、搜尋目的
如果不建立索引,在單查詢單個資料時稱做:線性查詢(無效率)
若設定索引後,稱作二位元搜尋(n log n)

順向、逆向索引(MySQL 8.0前版本沒有此功能)
主要在多欄位設定組合索引時有差別
單索引在查詢沒有順逆的差別

1、常拿來做搜尋條件的(必要設定索引)
如:name姓名、地址adderss、電話let
2、關連線的兩端都需要設定索引
3、查詢出來需要排序的結果需要設定索引

主索引(PK)主鍵、Primary Key
欄位名稱是底線或者欄位名稱前面*號也是主索引
不可重複、主索引不可空白不填
新增欄位若設定主索引預設索引
若主所有設定三個欄位時(複合索引),若搜尋其中單一或兩個欄位做搜尋時，這個索引則不會被查詢到,只有搜尋條件及順序要相同才會搜尋到,稱做索引前置

如設定主索引為 a b c欄位
只有查詢 a b c、a b、a才會搜尋到這個索引

參考索引(外來鍵、Foreign Key、FK)
維持資料正確性
在關連線的兩端設定參考索引,有方向性(單一方向性),多的則被少的參考,(基本上不設定、刪除單邊資料時較費時)
1、被參考的欄位需保證不重覆
解決方式
1、主索引
2、索引設定UNIQE代表資料不會重複

文本式沒有正規化

正規化
第一正規化不可少,每一個欄位只能放一個資料
至少設定到第二正規化
一對一可以合併成一個資料表
一對多可以拆分成兩個資料表
                                                                                                                                                                                                                                                     
多對多的關係 中間則需要第三個資料表(關聯)
兩個資料表的主索引拉出來做第三個資料表
